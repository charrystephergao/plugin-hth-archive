# plugin-hth-archive

本项目用于归档和发布多个独立的 HTML 页面。这些页面可以是工具、演示、文档或其它静态内容，不针对任何特定域名或网站。

## 项目简介

`plugin-hth-archive` 是一个简单的 HTML 页面归档仓库。它主要用于存放、组织和托管独立的网页文件，方便通过 GitHub Pages 或其他静态托管服务进行访问。每个页面独立存在，不依赖于特定后端服务或数据库。

## 目录说明

```
.
├── index.html          # 首页或导航页面（可选）
├── pages/              # 存放归档的 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有归档的 HTML 页面存放于此，每个文件对应一个独立页面。
- `assets/`：存放页面共享的样式、脚本和图像资源，便于统一维护。
- `index.html`：可选，可作为仓库的入口或导航页。

## 页面归档说明

- 每个 HTML 页面应自包含，尽量减少对外部资源的依赖。
- 如需使用公共资源，请将文件放入 `assets/` 目录，并在页面中引用相对路径。
- 页面命名建议使用小写字母、数字和连字符，避免空格和特殊字符。
- 归档页面不涉及任何具体网站或域名，仅作为独立内容展示。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建新的 HTML 文件，并确保所有引用资源路径正确。
- 更新资源：修改 `assets/` 下的文件时，注意检查所有引用该资源的页面是否需要同步更新。
- 删除页面：直接从 `pages/` 目录移除对应文件，同时删除 `index.html`（如有）中的相关链接。
- 建议保持目录结构清晰，定期清理不再使用的页面和资源。

## 部署

本仓库可通过 GitHub Pages 部署，将根目录或指定文件夹作为静态站点发布。具体部署方式请参考 GitHub 官方文档。
